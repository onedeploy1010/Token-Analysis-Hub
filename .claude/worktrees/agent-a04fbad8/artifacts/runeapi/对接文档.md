## 3. Community 合约接口

> 该合约用于维护用户的推荐人关系。根推荐人 `ROOT = 0x0000000000000000000000000000000000000001`。

### 3.1 referrerOf（查询推荐人）

- 类型：`view`（只读，不消耗 gas）
- 签名：

```solidity
function referrerOf(address account_) external view returns (address);
```

- 参数：
  - `account_`：要查询的用户地址。
- 返回值：
  - `address`：该用户的推荐人地址。
    - 返回 `0x0000...0000`（零地址）：表示该用户 **尚未绑定推荐人**。
    - 返回 `0x0000...0001`（ROOT）：表示该用户的推荐人是根节点（项目方初始地址）。
    - 返回其他地址：表示该用户的上级推荐人。
- 使用场景：
  - 进入 DApp 时，判断当前用户是否已绑定推荐人；未绑定时引导用户输入邀请人地址并调用 `addReferrer`。

### 3.2 addReferrer（绑定推荐人）

- 类型：`write`（需要用户签名、消耗 gas）
- 签名：

```solidity
function addReferrer(address referrer_) external;
```

- 参数：
  - `referrer_`：要绑定的上级推荐人地址。通常来自邀请链接中的参数，例如 `?ref=0x....`。
- 调用者：`msg.sender`（即当前登录钱包）即为被绑定推荐关系的用户。
- 校验规则（不满足会 revert）：
  1. `referrer_ != msg.sender` —— 推荐人不能是自己，否则报 `"Referrer is yourself"`。
  2. `referrer[msg.sender] == 0` —— 自己尚未绑定过推荐人，否则报 `"Referrer exists"`。
  3. `referrer[referrer_] != 0 || referrer_ == ROOT` —— 推荐人必须 **已经在社区里（即该推荐人自己也有推荐人）**，或者推荐人就是 ROOT，否则报 `"Referrer not invited"`。
- 事件：成功后触发 `EventAddReferrer(address indexed user, address referrer)`。

前端建议流程：
1. 进入页面时取 URL 中的 `ref` 参数，同时调用 `referrerOf(currentUser)` 判断是否已绑定。
2. 未绑定时：
   - 先调用 `referrerOf(ref)` 校验该邀请人本身是否已在社区内（返回非零地址）；ROOT 为例外（`0x...0001`）。
   - 调用 `addReferrer(ref)` 让用户签名。
3. 已绑定时：直接进入下一步业务。

## 4. NodePresell 合约接口

节点预售合约：用户使用 **USDT** 购买指定节点。关键约束：

- 每个地址 **只能购买一次**。
- 购买前 **必须先在 `Community` 合约绑定推荐人**（`referrerOf(msg.sender)` 非零地址），否则会 revert `ErrorNoBind()`。
- 每个节点配置了 **直推返佣比例 `directRate`**（基数 `PREVISION = 10000`，即 `10000 = 100%`）。支付时：
  - 若推荐人不是 ROOT（`0x...0001`）且 `directRate > 0`：`payAmount * directRate / 10000` 的 USDT 直接转给 **推荐人**，剩余部分转给项目方收款钱包 `RECEIVEWALLET`。
  - 若推荐人是 ROOT 或 `directRate == 0`：全部金额转给 `RECEIVEWALLET`。
- 合约本身不托管资金，全程 `transferFrom` 直达。

### 4.1 节点配置一览

节点配置由项目方初始化（见 `scripts/contractsHelper.ts` 中的 `Config_NodePresell_NodeConfigs`），当前配置如下：

| nodeId | 名称 | 单价（USDT） | 最大发售数量 | directRate（bps） | 直推返佣占比 |
| ------ | ---- | ------------ | ------------ | ----------------- | ------------ |
| 101    | 符主 | 50,000       | 40           | 1500              | 15%          |
| 201    | 符魂 | 10,000       | 200          | 1000              | 10%          |
| 301    | 符印 | 5,000        | 400          | 800               | 8%           |
| 401    | 符胚 | 2,500        | 800          | 500               | 5%           |

> 合约内部以 18 位精度存储金额，例如 `50,000 USDT` 对应 `50000 * 10^18`。
> `directRate` 单位是万分比（基数 `PREVISION = 10000`），`1500` 代表 `15%`。
> 实际可售余量 = `maxLimit - curNum`，请通过 `getNodeConfigs` 实时获取。

### 4.2 getNodeConfigs（查询节点配置与剩余）

- 类型：`view`
- 签名：

```solidity
struct NodeConfigRet {
    uint256 nodeId;     // 节点 ID
    address payToken;   // 支付代币（USDT 地址）
    uint256 payAmount;  // 单价（18 位精度）
    uint256 maxLimit;   // 最大发售数量
    uint256 curNum;     // 当前已售数量
    uint256 directRate; // 直推返佣比例，基数 10000（100% = 10000）
}

function getNodeConfigs(uint256[] calldata nodeIds_)
    external view
    returns (NodeConfigRet[] memory);
```

- 参数：
  - `nodeIds_`：要查询的节点 ID 数组，前端页面通常固定传 `[101, 201, 301, 401]`。
- 返回值：按入参顺序返回 `NodeConfigRet` 数组。
- 注意：
  - 若传入不存在的 `nodeId`，返回的 `payAmount` / `maxLimit` / `directRate` 均为 `0`，前端需过滤展示。
  - 直推返佣百分比 = `directRate / 100` %，例如 `1500` 即 `15%`。
- 使用场景：首页节点列表、价格、剩余名额、直推返佣展示。

### 4.3 getUserPurchaseData（查询用户购买记录）

- 类型：`view`
- 签名：

```solidity
function getUserPurchaseData(address user_)
    external view
    returns (
        address payToken,
        uint256 amount,
        uint256 payTime,
        uint256 nodeId
    );
```

- 参数：
  - `user_`：要查询的用户地址。
- 返回值：
  - `payToken`：支付的代币地址（一般为 USDT 地址）。
  - `amount`：支付数量（18 位精度）。
  - `payTime`：购买时间戳（秒）。
  - `nodeId`：购买的节点 ID。
- 用户未购买时，四个字段均为 `0` / 零地址，可用 `amount == 0` 作为「未购买」判定。
- 使用场景：
  - 进入「我的节点」页面展示订单。
  - 购买按钮 Disable 判断（每个地址只能购买一次）。

### 4.4 nodePresell（购买节点）

- 类型：`write`（需要用户签名、消耗 gas）
- 签名：

```solidity
function nodePresell(uint256 nodeId_) external;
```

- 参数：
  - `nodeId_`：要购买的节点 ID，取值 `101 / 201 / 301 / 401`。
- 校验规则（不满足会 revert）：
  1. **调用者必须已在 `Community` 合约绑定推荐人**（即 `Community.referrerOf(msg.sender) != address(0)`），否则 `ErrorNoBind()`。
  2. `nodeId_` 必须是有效节点，否则 `ErrorNodeId()`。
  3. 当前用户不能已购买过，否则 `ErrorPurchased()`。
  4. 当前节点剩余名额必须 > 0，否则 `"Limit"`。
  5. 用户 USDT 余额 >= `payAmount`，且已对 `NodePresell` 合约授权足够额度。
- 资金流（合约通过 `USDT.safeTransferFrom` 直达，**不托管资金**）：
  - 设 `directAmount = payAmount * directRate / 10000`。
  - 若推荐人不是 ROOT（`0x...0001`）且 `directRate > 0`：
    - `directAmount` → **推荐人地址**（`Community.referrerOf(user)`）
    - `payAmount - directAmount` → `RECEIVEWALLET`
  - 否则（推荐人为 ROOT 或 `directRate = 0`）：
    - 全部 `payAmount` → `RECEIVEWALLET`
- 事件：成功后触发

```solidity
event EventNodePresell(
    address indexed user,
    address payToken,
    uint256 amount,   // 购买总金额 = payAmount（非扣除返佣后的金额）
    uint256 time,
    uint256 num,      // 该节点成交后累计已售数量
    uint256 nodeId
);
```

#### 前端购买完整流程

1. 展示节点列表：调用 `getNodeConfigs([101, 201, 301, 401])` 展示单价、剩余名额、直推返佣比例。
2. 用户点击某节点的「购买」按钮：
   1. 调用 `Community.referrerOf(user)`，若返回 `0x0` 则拦截购买，先引导用户绑定推荐人（见 §3.2）。
   2. 调用 `getUserPurchaseData(user)`，若 `amount > 0`，提示「已购买过」。
   3. 调用 `USDT.allowance(user, NodePresell)`，若小于 `payAmount`：
      - 调用 `USDT.approve(NodePresell, payAmount)`（建议直接 approve `type(uint256).max` 避免重复授权），等待交易确认。
   4. 调用 `NodePresell.nodePresell(nodeId)`，等待交易确认。
3. 监听 `EventNodePresell` 或刷新 `getUserPurchaseData` 以更新 UI。
4. 可选：给推荐人端展示返佣收入，可通过监听 USDT 的 `Transfer` 事件（`from = 购买者`, `to = 推荐人`）或 `EventNodePresell` + `getNodeConfigs` 计算 `directAmount`。
